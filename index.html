PRICIPAIS REQISITOS DO MEU SISTEMA BIBILIOTECA

  • Cadastro de Livros Adicionar informações detalhadas sobre os livros, como título, autor, editora, ISBN, número de páginas.
  • Cadastro de Leitores Permitir o registro de informações sobre os leitores, como nome, endereço, e-mail, telefone.
  • Controle de Empréstimos Registrar empréstimos de livros para os leitores, associando as datas de retirada e devolução.
  • Consulta de Livros Oferecer uma interface de busca eficiente para que os usuários possam encontrar livros facilmente.
  • Notificações por E-mail Enviar lembretes de datas de devolução, confirmações de reservas, etc., por e-mail
  • Integração com Site Disponibilizar um site amigável para os leitores acessarem o catálogo, realizarem pesquisas e gerenciarem suas contas.
  • Integração com Computadores Desenvolver uma versão de desktop do sistema para ser usada localmente na biblioteca, se necessário.
  • Acesso pela Internet Certificar-se de que o sistema seja acessível pela internet para facilitar o gerenciamento remoto.

Implementação do CRUD no Sistema

  • Create (Criar) Cadastro de Livros: Adicionar novos livros ao sistema, incluindo informações como título, autor, número de cópias disponíveis.
  • Read (Ler) Consulta de Livros: Permitir que os usuários busquem e vejam informações sobre livros disponíveis na biblioteca.
  • Update (Atualizar) Atualização de Dados do Livro: Modificar informações de livros existentes, como quantidade disponível ou detalhes do autor
  • Delete (Excluir) Remoção de Livros Obsoletos: Excluir registros de livros que não estão mais disponíveis na biblioteca.

RELAÇAO ENTRE LIVRO USUARIO E EMPRESTIMO

  • Livro Representa as informações específicas de um livro, como título, autor, número de cópias disponíveis.
  • Usuário Representa um usuário do sistema, como um membro da biblioteca.
  • Empréstimo Representa a transação de empréstimo de um livro para um usuário em específico.
